## 项目概述

DeepSeek 安全评估平台是一款集成了网络扫描、威胁情报分析、自动防御、病毒查杀等功能的综合性安全评估工具。本平台旨在帮助用户全面评估网络安全状况，及时发现和修复安全漏洞，提高网络系统的整体安全性。

主要功能特点：
- 自动化网络扫描，发现网络资产和开放端口
- 智能安全风险评估和报告生成
- 基线合规性检查
- 威胁情报分析和态势感知
- 自动化防御策略实施
- 文件和哈希病毒查杀
- 模型投毒检测
- 边缘设备管理和监控

## 快速开始

### 系统要求

- Node.js 14.x 或更高版本
- npm 6.x 或更高版本
- 现代浏览器（Chrome、Firefox、Safari、Edge等）
- 足够的网络权限进行扫描操作

### 安装与启动

1. 克隆或下载项目代码到本地
2. 进入server目录安装依赖：
   ```
   cd server
   npm install
   ```
3. 启动后端服务：
   ```
   npm start
   ```

### 访问方式

服务启动后，通过浏览器访问：http://localhost:8787

## 用户管理

### 登录系统

DeepSeek 安全评估平台使用基于 JWT 的身份验证机制，所有后续 API 请求都会使用登录时获取的 token 进行身份验证。

**操作步骤：**
1. 在浏览器中访问 http://localhost:8787，自动进入登录页面
2. 在登录页面输入您的用户名和密码
3. 点击「登录」按钮进行身份验证
4. 登录成功后，系统会自动跳转到仪表盘页面
5. 系统会将认证 token 存储在浏览器的本地存储中，在会话期间保持登录状态

**默认账户：**
- 首次使用时，默认管理员账户可能需要通过系统配置初始化
- 管理员密码可以在登录后通过「配置」页面进行修改

**登录失败处理：**
- 如果输入的用户名或密码不正确，登录状态区域会显示「登录失败」提示
- 请检查您的输入是否正确，并确保账户已完成注册

### 注册账户

**操作步骤：**
1. 在登录页面点击 URL 中的「#/register」链接或直接访问 http://localhost:8787/#/register
2. 在注册页面输入用户名、密码和确认密码
3. 系统会在前端验证密码一致性，如果两次输入的密码不一致，会显示「请检查用户名与密码输入」提示
4. 点击「注册」按钮完成账户创建
5. 注册成功后，系统会显示「注册成功，请登录」提示，并自动跳转至登录页面
6. 使用新创建的账户凭据进行登录

**注意事项：**
- 用户名不应包含空格，系统会自动去除首尾空格
- 密码强度要求取决于系统配置
- 建议使用强密码并定期更换以确保账户安全

## 仪表盘功能

仪表盘是系统的主要工作界面，提供了网络扫描、资产概览、安全报告和修复计划等核心功能。登录成功后，系统将自动跳转到仪表盘页面。

### 资产概览

仪表盘顶部展示了资产统计信息：

- **资产总数**：显示当前扫描发现的所有资产数量
- **高危资产**：显示被评估为高风险的资产数量
- **中危资产**：显示被评估为中等风险的资产数量
- **低危资产**：显示被评估为低风险的资产数量

### 网络扫描

网络扫描功能允许用户对指定的网络范围进行扫描，发现开放端口和潜在安全风险。

**操作步骤：**

1. 在CIDR输入框中输入要扫描的网络地址范围（如：192.168.0.0/24）
2. **可选参数配置：**
   - **端口范围**：在端口输入框中指定要扫描的端口列表，多个端口用逗号或空格分隔（如：22,80,443,8080）
   - **AI模型选择**：在模型选择器中选择要使用的AI模型，用于生成安全评估报告
3. 点击「开始扫描」按钮启动扫描任务
4. 扫描过程中，状态显示区域会显示「扫描进行中」
5. 扫描完成后，状态显示区域会更新为「完成」

**扫描结果展示：**

- **资产表格**：显示扫描发现的主机、开放端口和风险等级
- **端口分布图**：展示各端口的资产分布情况
- **安全评估报告**：基于AI分析生成的详细安全评估内容
- **威胁情报**：集成多个威胁情报源的信息，提供关于发现资产的安全情报

### 安全报告

扫描完成后，系统会自动生成全面的安全评估报告，包括：

- 网络资产概览
- 开放端口分析
- 潜在安全风险评估
- 详细的安全建议

**报告导出功能：**

1. 扫描完成后，点击「打印报告」按钮
2. 系统会打开新窗口显示格式化的报告内容
3. 使用浏览器的打印功能，可以将报告保存为PDF或直接打印

### 修复计划

基于扫描结果，系统可以生成详细的修复计划，指导用户如何解决发现的安全问题。

**操作步骤：**

1. 确保已完成至少一次网络扫描
2. 点击「生成修复计划」按钮
3. 系统将基于最新的扫描结果生成修复建议
4. 修复计划将显示在专用区域，包含详细的修复步骤和优先级建议

**注意事项：**

- 扫描结果会被系统保存在会话中，用户可以随时查看和导出
- 扫描过程可能需要一定时间，取决于网络规模和指定的端口范围
- 建议在非工作时间进行大规模扫描，以避免对网络性能造成影响

## 基线排查

基线排查功能帮助用户检查系统配置是否符合安全基线标准，识别潜在的配置问题和安全漏洞。

### 基线策略选择

**操作步骤：**

1. 从导航栏访问「基线排查」页面
2. **基线策略配置：**
   - 从下拉菜单中选择基线策略标准，包括CIS Level 1、CIS Level 2、ISO 27001或自定义策略
   - 勾选「使用最近扫描结果」复选框，可以直接基于最新的网络扫描结果进行基线排查
3. 点击「开始基线排查」按钮启动排查任务
4. 排查过程中，状态显示区域会显示「排查进行中」
5. 排查完成后，状态显示区域会更新为「完成」

### 基线报告生成

基线排查完成后，系统会自动生成详细的基线合规性报告：

- **合规状态分析**：显示整体合规率和不合规项数量
- **详细合规检查结果**：列出每一项基线检查的具体结果，包括合规项和不合规项
- **不合规项风险评估**：对发现的不合规项进行风险等级划分
- **修复建议**：针对每一项不合规问题提供具体的修复建议和最佳实践

**报告导出功能：**

1. 排查完成后，点击「打印基线报告」按钮
2. 系统会打开新窗口显示格式化的报告内容
3. 使用浏览器的打印功能，可以将报告保存为PDF或直接打印

## 态势感知

态势感知功能帮助用户实时了解当前网络安全态势，集成多个威胁情报源，提供全面的安全态势分析。

### 数据源配置

态势感知功能集成了多个外部威胁情报源，提供更全面的安全态势分析：

- **AlienVault OTX**：提供全球威胁情报和新兴威胁信息
- **CISA KEV (Known Exploited Vulnerabilities)**：美国网络安全和基础设施安全局的已知被利用漏洞目录
- **Shodan**：互联网设备搜索引擎，提供端口和服务信息
- **AbuseIPDB**：提供IP地址滥用报告和评分
- **VirusTotal**：提供文件和URL的恶意软件检测结果

**配置方法：**

在系统配置页面（#/config）中设置相关API密钥，以获取更详细的威胁情报信息。

### 态势分析

**操作步骤：**

1. 从导航栏访问「态势感知」页面
2. 点击「开始态势分析」按钮启动分析任务
3. 分析过程中，状态显示区域会显示「分析进行中」
4. 分析完成后，状态显示区域会更新为「完成」

**分析结果展示：**

- **威胁情报源概览**：
  - OTX Trending：显示当前流行的威胁情报主题和标签
  - CISA KEV：显示最新的已知被利用漏洞信息

- **资产情报**：
  - 显示系统扫描发现的资产相关的威胁情报
  - 包括IP地址的VT声誉评分、OTX脉冲数量、Shodan开放端口和AbuseIPDB评分等

- **态势分析报告**：
  - 基于收集的所有情报生成综合分析报告
  - 提供当前网络安全态势的评估和建议

**报告导出功能：**

1. 分析完成后，点击「打印态势报告」按钮
2. 系统会打开新窗口显示格式化的报告内容
3. 使用浏览器的打印功能，可以将报告保存为PDF或直接打印

**注意事项：**

- 态势分析的深度和准确性取决于配置的外部情报源API密钥
- 某些高级情报功能可能需要对应的情报源账户或订阅
- 建议定期进行态势分析，以保持对最新威胁的了解

## 自动化防御

自动化防御功能提供无人值守的安全防御能力，可以按照设定的周期自动执行安全审查和防御操作。

### 防御策略配置

**操作步骤：**

1. 从导航栏访问「自动化防御」页面
2. **防御参数配置：**
   - **防御周期**：在输入框中设置防御执行的间隔时间（秒）
   - **无人值守模式**：勾选复选框启用无人值守模式，系统将自动应用防御措施
3. 点击「启动防御」按钮开始自动防御任务
4. 系统将根据设定的周期自动执行安全审查和防御操作
5. 需要停止自动防御时，点击「停止防御」按钮

### 防御状态监控

系统提供实时的防御状态监控，包括：

- **防御状态**：显示当前防御系统是否已启用
- **上次运行时间**：显示最近一次防御操作的执行时间
- **防御报告**：显示详细的防御分析报告和结论
- **已应用的防御动作**：列出所有已应用的防御措施及其应用时间

**报告导出功能：**

1. 点击「打印防御报告」按钮
2. 系统会打开新窗口显示格式化的防御报告内容
3. 使用浏览器的打印功能，可以将报告保存为PDF或直接打印

**注意事项：**

- 防御周期设置建议：对于资源有限的环境，建议设置较长的防御周期（如3600秒或更长）
- 无人值守模式可能会对系统配置进行自动修改，请确保了解潜在影响
- 建议在启用自动防御前，先进行测试以确保防御策略不会对正常业务造成影响

## 病毒查杀

病毒查杀功能提供多维度的恶意代码检测能力，包括文件上传查杀、哈希查杀和模型投毒检测。

### 文件上传查杀

通过上传文件进行实时恶意代码分析。

**操作步骤：**

1. 从导航栏访问「病毒查杀」页面
2. 点击「选择文件」按钮，从本地文件系统选择待检测的文件
3. 点击「上传并查杀」按钮开始文件分析
4. 分析过程中，状态显示区域会显示「查杀进行中」
5. 分析完成后，状态显示区域会更新为「完成」

**查杀结果展示：**

- **病毒家族**：识别出的恶意软件家族（如适用）
- **VirusTotal统计**：显示多个杀毒引擎的检测结果统计
- **详细分析报告**：基于AI分析生成的详细恶意代码分析报告

### 哈希查杀

通过文件哈希值（SHA256）进行快速恶意代码检测，无需上传整个文件。

**操作步骤：**

1. 在「病毒查杀」页面的哈希输入框中，输入文件的SHA256哈希值
2. 点击「按哈希查杀」按钮开始分析
3. 分析过程中，状态显示区域会显示「查杀进行中」
4. 分析完成后，状态显示区域会更新为「完成」

**查杀结果展示：**

- 与文件上传查杀相同的结果格式
- 系统会基于哈希值查询多个威胁情报源，获取最新的检测信息

### 模型投毒检测

检测文本输入中是否包含模型投毒攻击的特征。

**操作步骤：**

1. 在「病毒查杀」页面的文本输入框中，输入需要检测的可疑提示或文本
2. 点击「检测投毒」按钮开始分析
3. 分析结果会直接显示在文本区域下方

**检测结果说明：**

- 系统会分析输入文本中是否包含针对AI模型的投毒攻击特征
- 提供详细的风险评估和建议

### 病毒统计信息

页面还提供了病毒查杀的统计信息：

- 显示常见病毒家族及其出现次数的排行榜
- 帮助用户了解当前流行的恶意软件趋势

**报告导出功能：**

1. 查杀完成后，点击「打印报告」按钮
2. 系统会打开新窗口显示格式化的病毒分析报告内容
3. 使用浏览器的打印功能，可以将报告保存为PDF或直接打印

**注意事项：**

- 文件上传大小可能受到系统限制，请查看系统配置
- 哈希查杀需要精确的SHA256哈希值
- 模型投毒检测主要针对AI模型安全，帮助识别潜在的提示注入或数据投毒攻击

## 边缘设备管理

边缘设备管理功能允许管理员集中管理、监控和控制连接到系统的各类边缘安全设备，提供全面的设备生命周期管理能力。

### 设备列表管理

**设备列表展示：**

1. 从导航栏访问「边缘设备管理」页面
2. 页面将显示所有已注册和连接的边缘设备列表
3. 设备列表包含以下关键信息：
   - **设备ID**：边缘设备的唯一标识符
   - **设备名称**：用户自定义的设备名称
   - **IP地址**：设备的网络地址
   - **连接状态**：显示设备是「在线」还是「离线」
   - **最后在线时间**：设备最近一次活动的时间戳
   - **设备类型**：设备的硬件或软件类型分类
   - **版本信息**：设备固件或软件版本
   - **操作状态**：当前设备的运行状态

**设备搜索与过滤：**

1. 使用搜索框可以根据设备ID、名称或IP地址快速查找特定设备
2. 支持按设备类型、连接状态等条件进行过滤，方便管理大量设备

### 设备操作管理

**添加新设备：**

1. 点击「添加设备」按钮
2. 在弹出的对话框中填写设备信息：
   - 设备ID（必填，系统将生成唯一ID或允许手动输入）
   - 设备名称（必填，方便用户识别）
   - IP地址（必填，设备的网络地址）
   - 设备类型（必填，从下拉列表中选择）
   - 描述（可选，设备的详细说明）
3. 点击「保存」按钮完成设备添加
4. 系统将自动尝试连接并初始化新添加的设备

**编辑设备信息：**

1. 在设备列表中找到需要修改的设备
2. 点击该设备对应的「编辑」按钮
3. 在弹出的编辑对话框中修改相关信息
4. 点击「保存」按钮更新设备信息

**移除设备：**

1. 在设备列表中找到需要移除的设备
2. 点击该设备对应的「移除」按钮
3. 在弹出的确认对话框中确认移除操作
4. 系统将断开与设备的连接并从列表中移除

**远程控制功能：**

系统支持对在线边缘设备进行远程控制操作：

1. 在设备列表中找到目标在线设备
2. 点击该设备对应的「控制」按钮
3. 选择需要执行的远程操作：
   - 重启设备：远程重启边缘设备
   - 更新配置：将最新配置应用到设备
   - 启动/停止服务：控制设备上运行的安全服务
   - 推送更新：推送最新固件或软件更新包到设备

### 设备监控与告警

**实时状态监控：**

系统提供边缘设备的实时状态监控功能：

1. 设备列表中的连接状态指示器显示设备在线/离线状态
2. 可以查看设备的实时性能指标：
   - CPU使用率
   - 内存使用率
   - 磁盘空间
   - 网络流量
   - 安全服务状态

**告警配置与通知：**

1. 可以为每个设备设置个性化的告警阈值：
   - CPU使用率超过阈值告警
   - 内存使用率超过阈值告警
   - 设备离线时长超过阈值告警
   - 安全事件检测告警
2. 告警通知方式可配置，支持：
   - 系统内通知
   - 邮件通知
   - 第三方集成（如Webhook）

### 设备分组与批量操作

对于大量设备的管理，系统提供分组管理和批量操作功能：

1. 创建设备分组，将设备按功能、位置或重要性分类
2. 对分组内设备执行批量操作：
   - 批量配置更新
   - 批量服务控制
   - 批量软件更新

**注意事项：**

- 确保设备的IP地址在网络中可访问，且已正确配置防火墙规则，允许系统与设备通信
- 新添加的设备需要在设备端安装相应的客户端或代理软件
- 远程控制操作可能会影响设备的正常运行，请谨慎操作
- 建议定期检查设备连接状态和性能指标，确保系统整体安全防护能力

## 常见问题和故障排除

本节提供系统使用过程中可能遇到的常见问题及其解决方案，帮助用户快速排查和解决各类技术问题。

### 账户和登录问题

**登录失败排查：**

1. **凭据错误**
   - 检查用户名和密码是否输入正确，注意大小写
   - 确保密码未包含多余的空格
   - 尝试使用「忘记密码」功能重置密码

2. **账户锁定**
   - 症状：收到「账户已锁定」错误消息
   - 原因：多次密码输入错误导致系统自动锁定
   - 解决方案：联系系统管理员解锁账户，或等待锁定时间自动过期（默认30分钟）

3. **会话超时**
   - 症状：页面显示「会话已过期」，要求重新登录
   - 原因：长时间未活动触发了会话超时机制
   - 解决方案：重新登录系统，并调整浏览器设置以保持会话活跃

**权限相关问题：**

1. **无法访问特定功能**
   - 症状：某些菜单项或按钮显示为灰色或完全隐藏
   - 原因：当前用户权限不足
   - 解决方案：联系系统管理员，请求分配相应权限

2. **操作被拒绝**
   - 症状：执行操作时收到「权限不足」错误消息
   - 原因：当前用户没有执行该操作的权限
   - 解决方案：确认您是否使用了正确的账户角色，或联系管理员提升权限

### 系统功能问题

**扫描功能故障：**

1. **扫描任务无法启动**
   - 症状：点击「开始扫描」按钮后无响应或显示错误
   - 可能原因：
     - 输入的IP地址格式不正确
     - 目标网络不可访问
     - 系统资源不足
   - 解决方案：
     - 检查IP地址格式是否正确（特别是CIDR格式）
     - 验证目标网络是否可达（尝试ping或traceroute）
     - 关闭其他占用资源的程序，释放系统资源

2. **扫描速度异常缓慢**
   - 症状：扫描任务执行时间超出预期
   - 可能原因：
     - 扫描范围过大
     - 系统资源不足
     - 网络带宽限制
   - 解决方案：
     - 减少扫描范围或分批扫描
     - 增加系统内存或CPU资源
     - 调整扫描参数，降低并发请求数

3. **扫描结果不准确**
   - 症状：发现明显的误报或漏报
   - 可能原因：
     - 扫描规则库过旧
     - 防火墙或安全组阻止了部分扫描请求
     - 扫描深度设置不足
   - 解决方案：
     - 更新系统规则库
     - 检查网络防火墙设置，确保扫描流量可通过
     - 增加扫描深度，启用更全面的检查选项

**防御功能故障：**

1. **自动防御无法启动**
   - 症状：点击「启动防御」按钮后无响应或显示错误
   - 可能原因：
     - 权限不足（需要管理员权限）
     - 系统服务未运行
   - 解决方案：
     - 确保使用管理员账户登录
     - 检查并启动必要的系统服务

2. **防御措施未生效**
   - 症状：防御已启用，但未检测到或阻止预期的威胁
   - 可能原因：
     - 防御规则配置不当
     - 防御模式设置不正确
   - 解决方案：
     - 检查并调整防御规则配置
     - 确保启用了「无人值守模式」以自动应用修复

### 性能和资源问题

**系统性能优化：**

1. **系统运行缓慢**
   - 症状：界面响应延迟，操作卡顿
   - 可能原因：
     - 系统负载过高
     - 数据库性能问题
     - 浏览器缓存问题
   - 解决方案：
     - 检查并关闭不必要的后台任务
     - 优化数据库（如定期清理历史数据）
     - 清除浏览器缓存和Cookie，尝试使用不同的浏览器

2. **内存占用过高**
   - 症状：系统显示内存不足警告，或出现应用崩溃
   - 可能原因：
     - 同时运行多个大型扫描任务
     - 内存泄漏
   - 解决方案：
     - 避免同时启动多个大型任务
     - 重启应用服务释放内存
     - 增加服务器内存配置

3. **磁盘空间不足**
   - 症状：系统提示磁盘空间不足，无法保存数据
   - 可能原因：
     - 日志文件过大
     - 历史扫描结果过多
     - 临时文件未清理
   - 解决方案：
     - 清理或归档过期的日志文件
     - 设置自动删除超期的扫描结果
     - 清理临时目录中的文件

### 网络和连接问题

**连接问题排查：**

1. **API连接失败**
   - 症状：功能正常但无法连接外部API服务
   - 可能原因：
     - 网络连接问题
     - API密钥无效
     - 防火墙阻止了出站连接
   - 解决方案：
     - 检查网络连接和DNS解析
     - 验证API密钥是否有效且未过期
     - 检查防火墙设置，确保允许相关出站连接

2. **边缘设备连接问题**
   - 症状：边缘设备显示为离线状态
   - 可能原因：
     - 设备网络连接中断
     - 设备防火墙设置阻止了通信
     - 设备代理配置错误
   - 解决方案：
     - 检查设备的网络连接
     - 确认设备防火墙允许与系统通信的端口
     - 验证代理配置是否正确

### 日志和诊断

**使用日志进行故障排查：**

1. **系统日志查看**
   - 访问「系统日志」页面查看详细的系统运行日志
   - 使用过滤器按级别（错误/警告/信息）或时间范围筛选日志
   - 查找与问题相关的错误消息和堆栈跟踪

2. **诊断模式**
   - 遇到复杂问题时，可启用「诊断模式」获取更详细的日志信息
   - 完成故障排查后记得关闭诊断模式，以避免过多的日志占用磁盘空间

**联系支持前的准备工作：**

当您无法自行解决问题时，请联系技术支持团队，并准备以下信息：

1. 系统版本信息（从「关于」页面获取）
2. 相关错误消息的截图
3. 系统日志文件（特别是错误发生时间段的日志）
4. 详细描述问题的复现步骤
5. 问题发生前的任何系统更改（如配置修改、更新等）

### 备份和恢复建议

为了确保系统数据安全，强烈建议定期执行以下操作：

1. 定期备份系统配置和重要数据
2. 在进行重大配置更改前创建配置备份
3. 定期测试备份的可恢复性，确保在需要时能够正常恢复
4. 建立应急恢复计划，明确在系统故障时的恢复流程和责任人
